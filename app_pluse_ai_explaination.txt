🚀 APP Pulse AI — Complete Flow Explanation

Your project is now divided into modules.

Each module has ONE responsibility.

This is exactly how production AI systems are designed.

📁 Overall Architecture
User Uploads ODS File
        ↓
Preprocessing
        ↓
Guardrails Validation
        ↓
AI Agents Analysis
        ↓
RAG Embedding + Retrieval
        ↓
Charts + Insights + Chatbot
📄 App_Pulse_AI.py

This is the main application.

Think of it as:

the controller
dashboard
orchestrator

It connects EVERYTHING.

✅ What it does
1. Uploads ODS file
uploaded_file = st.file_uploader(...)

User uploads performance metrics.

Example:

Time	CPU %	Memory %	API Latency
2. Loads dataframe
df = pd.read_excel(file, engine="odf")

Reads ODS spreadsheet into pandas.

3. Cleans data
df = clean_dataframe(df)

Removes:

spaces
duplicates
formatting issues
4. Converts columns to numeric
df = convert_numeric_columns(df)

Without this:

charts fail
ML models fail
anomaly detection breaks
5. Validates dataset
missing_cols = validate_dataframe(df)

Checks:

Time column exists
required fields exist

This is a guardrail.

6. Creates vector database
create_vector_store(df)

This powers RAG.

Converts your metrics into embeddings.

7. Runs AI agents
performance_agent.analyze(df)

Each agent performs a specialized task.

8. Displays insights
st.warning(...)

Shows:

CPU spikes
memory issues
deployment correlations
9. Detects anomalies
IsolationForest()

Machine learning identifies unusual rows.

10. Creates charts
px.line(...)

Visualizes metrics over time.

11. Runs RAG chatbot
retriever.get_relevant_documents(query)

Searches relevant performance data using embeddings.

📁 rag/

This folder handles RAG.

RAG = Retrieval-Augmented Generation.

📄 embeddings.py
Purpose

Loads embedding model.

HuggingFaceEmbeddings()
✅ What are embeddings?

Embeddings convert text into vectors.

Example:

"CPU spike"
      ↓
[0.123, 0.882, 0.292, ...]

This allows semantic search.

📄 ingest.py
Purpose

Creates vector database.

Flow
Dataframe
   ↓
Rows converted to text
   ↓
Text split into chunks
   ↓
Embeddings generated
   ↓
Stored in ChromaDB
Example

Row:

Time: 10:05 | CPU: 95 | Event: Deployment

Gets stored as vector embedding.

📄 retriever.py
Purpose

Searches vector DB.

Example

User asks:

"Why did latency increase?"

Retriever finds similar rows:

deployment events
high latency rows
anomalies

📄 performance_agent.py
Purpose

What it checks

📄 anomaly_agent.py
Purpose

Machine learning anomaly detection.

Uses:

IsolationForest

Why important?

Real production systems:

leak memory
spike CPU

Anomaly detection catches hidden issues.
